Binary Search Tree (BST) Structure

Understanding the fundamental properties and terminology

BST Property: For every node:
  • All values in the left subtree are less than the node's value
  • All values in the right subtree are greater than the node's value

Key Terminology

  • Root: Topmost node, has no parent
  • Parent: Node with children below it
  • Child: Node with a parent above it
  • Leaf: Node with no children (0 children)
  • Subtree: Tree formed by a node and its descendants

Node Relationships

  • Every node (except root) has exactly 1 parent
  • Each node can have 0, 1, or 2 children
  • Left child: value < parent
  • Right child: value > parent
  • No duplicates are stored in BST

BST Insertion Process

Step-by-step guide to inserting values into a Binary Search Tree

Insertion Sequence:
50, 20, 90, 19, 41
Starting with empty tree
The tree is empty. Ready to insert the first node.
Insertion Algorithm:
  1. Always start at the root
  2. Compare value with current node:
    • If value < current → go left
    • If value > current → go right
  3. Insert at the first null position you encounter
  4. Insert values in the exact order given

BST Deletion Cases

Three different scenarios when deleting a node from a BST

Case 1: Deleting a Leaf Node (0 children)
Node has no children
Simply set parent's link to null
This is the simplest case - no reconnection needed

Case 1: No Children (Leaf)

Simply remove the node by setting parent's pointer to null

Case 2: One Child

Connect parent directly to the single child (bypass the deleted node)

Case 3: Two Children

Replace with in-order successor: go right once, then all the way left

Alternative for Case 3

Can also use in-order predecessor: go left once, then all the way right

Matrix is symmetric. 1 = edge exists, 0 = no edge.

Key Matrix Properties:
  • Undirected graphs: Matrix is symmetric (row i,j = row j,i)
  • Directed graphs: Matrix is NOT symmetric
  • Unweighted: Use 1 for edge exists, 0 or ∞ for no edge
  • Weighted: Use actual weight values, ∞ for no edge
  • Diagonal: Always 0 (no self-loops)
  • Order: Matrix must be in alphabetical order

Depth-First Search (DFS)

Goes as deep as possible before backtracking - uses a stack

Important: When processing edges, follow the order specified in the question:
  • Alphabetical order (most common)
  • Decreasing weight order (for weighted graphs)
DFS Algorithm:
1. Start at vertex A, mark as visited
2. Process edges in decreasing weight order: F(75) is highest
3. Visit F, then continue to next highest unvisited neighbor
4. Continue until all reachable vertices are visited
Output: A F D E C B
DFS Characteristics:
  • Uses a stack (or recursion)
  • Goes deep before exploring siblings
  • Time complexity: O(V + E)
  • Space complexity: O(V) for visited array

Breadth-First Search (BFS)

Visits all neighbors at current level before moving deeper - uses a queue

Important: Process neighbors in alphabetical order when adding to queue.
BFS Algorithm:
1. Add starting vertex C to queue
2. Dequeue C, visit it, add all neighbors (A,B,E) in alphabetical order
3. Dequeue A, visit it, add unvisited neighbors to queue
4. Continue until queue is empty
Queue: [C]
Output: C A B E G F D
BFS Characteristics:
  • Uses a queue
  • Visits level by level (breadth-first)
  • Finds shortest path in unweighted graphs
  • Time complexity: O(V + E)
  • Space complexity: O(V) for queue

Exam Tips

  • BST Insertion: Always insert in the exact order given, start at root for each insertion
  • BST Traversals: In-order gives sorted output, memorize the patterns
  • BST Deletion: Know all 3 cases, practice finding in-order successor
  • Adjacency Matrix: Must be in alphabetical order, know symmetric vs non-symmetric
  • DFS: Stack-based, process edges as specified (alphabetical or by weight)
  • BFS: Queue-based, always process neighbors in alphabetical order
  • Practice: Work through examples step-by-step, draw diagrams
