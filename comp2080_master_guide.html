📚 BST Foundations & Theory

A Binary Search Tree (BST) is a hierarchical data structure that allows for efficient searching, insertion, and deletion operations. Its power comes from the fundamental **BST Property**.

The BST Property: For every node in the tree:
  • All values in the Left Subtree are strictly **less than** the node's value.
  • All values in the Right Subtree are strictly **greater than** the node's value.

Core Terminology

Before implementing algorithms, you must understand the relational terms:

  • Root: The starting node (top). No parent.
  • Parent: A node that has links to lower nodes.
  • Child: A node pointed to by a higher node.
  • Leaf: A node with zero children (end of path).
  • Internal Node: A node with at least one child.
  • Height: The longest path from root to any leaf.

Performance Metrics

On average, operations in a balanced BST are highly efficient:

  • Search: O(log n)
  • Insert: O(log n)
  • Delete: O(log n)
Worst Case: If nodes are inserted in sorted order (1, 2, 3...), the tree becomes a "Line" (Skewed), and performance degrades to O(n).

📥 BST Insertion: Step-by-Step

Insertion is a recursive process. You start at the root and compare the new value to the current node to decide which branch to take.

The Algorithmic Logic:
  1. If the tree is empty, new value becomes **ROOT**.
  2. If New < Current, go to the LEFT subtree.
  3. If New > Current, go to the RIGHT subtree.
  4. Repeat until you hit a **NULL** link, then place the node there.

🔍 BST Traversal Visit sequences

Traversing means visiting every node in the tree exactly once. The order you choose determines the logic of the visit.

In-Order Traversal

Rule: (Left Subtree → Root → Right Subtree)

This is the most powerful visit for a BST because it yields values in Sorted Ascending Order.

♻️ BST Deletion: The 3 Scenarios

Deleting a node requires restructuring the links. There are 3 distinct cases based on the number of children the target has.

Case 1: No Children

Simply remove the link from the parent to this node. The node is orphaned and cleared.

The Case 3 Successor Rule: To maintain the BST property, we replace the target with its **In-Order Successor** (smallest in the right subtree). This successor is guaranteed to be a Case 1 or Case 2 node, making it easy to remove after the swap.

🛠️ Construction Algorithm: Rebuilding Trees

You can recreate a tree if you have the In-Order sequence and either a Pre-Order or Post-Order sequence.

The "Split" Trick: 1. Pre-order's first node is the **Root**. 2. Find this root in the In-order list. 3. Everything to the **left** of it is the Left Subtree. 4. Everything to the **right** is the Right Subtree. 5. Repeat recursively.

🕸️ Graph Search Algorithms: DFS & BFS

Depth-First and Breadth-First search differ in how they expand from the source node.

DFS: Depth-First

Rule: Uses a **Stack**. Branch as deep as possible.

Priority: Adjacent nodes added to stack in reverse alphabetical order (so A is popped first).

BFS: Breadth-First

Rule: Uses a **Queue**. Explore all neighbors at current level first.

Priority: Neighbors added to queue in alphabetical order.

The Node Class

class Node {
    int data;
    Node left, right;
    
    public Node(int val) {
        data = val;
        left = right = null;
    }
}

Full Insertion Implementation

public void insert(int v) {
    root = insertRec(root, v);
}

private Node insertRec(Node cur, int v) {
    if (cur == null) return new Node(v);
    if (v < cur.data) cur.left = insertRec(cur.left, v);
    else if (v > cur.data) cur.right = insertRec(cur.right, v);
    return cur;
}

Full Deletion (3-Case logic)

public void delete(int x) {
    root = deleteRec(root, x);
}
